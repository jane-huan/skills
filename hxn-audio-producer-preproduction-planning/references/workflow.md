# Workflow

> Version: v0.6.9
> Status: 正式发布，持续迭代

## 1. 目标

把一本书从“拿到可用原文”推进到“可以开始稳定录制”的前期筹备状态。

## 2. 输入前置物

本 skill 接收可用原文作为输入。可用原文可以是用户直接提供的正文，也可以是其他流程整理后的原文整理稿。本 skill 不负责底层 OCR、清洗或错字修复；发现原文疑点时记录到“原文问题与制作修订记录”。

## 3. 主流程

```text
轻量项目初始化
→ 阅读理解块留痕
→ 资料包滚动积累
→ 全书资料包 v1
→ 角色分析工作簿
→ 集数规划
→ 角色颜色体系
→ 下一步生产动作判断
   ├─ 分集文本 → 原文画本
   │              ├─ 可告一段落
   │              ├─ 可继续进入录制执行稿
   │              └─ 可继续进入录制版画本
   └─ 分集文本 → 录制执行稿
                  ├─ 可告一段落
                  └─ 可继续进入录制版画本
→ 试录 / 第 1 集校准
→ 后续集数按校准值估算
```

集数规划之后不默认进入原文画本，也不默认进入录制执行稿。AI 应先根据平台、交付要求、制作人详细要求和是否需要录制方使用画本，判断下一步生产动作。

原文画本与录制执行稿是相对独立的生产节点，不是固定前后级关系：

- 可以直接由分集文本生成录制执行稿，再决定是否继续进入录制版画本。
- 可以先做原文画本，再决定是否继续进入录制执行稿或录制版画本。
- 可以只完成其中一项，并把本轮共创停在该产物。
- 如果先完成其中一项，后续再做另一项时，应承认上一项成果存在，并以它作为可回查输入之一，不要重置流程。
- 如果平台或软件可基于整理后的文本自动画本，且本轮共创目标只是完成可录、可演、可剪的文字底稿，可以停在录制执行稿。

## 4. 轻量项目初始化记录

项目初始化记录是新书启动时的制作参数卡，不是一次性填完的大问卷。AI 应主动提醒并建立一版轻量记录：已知信息先落地，未知信息标“待确认”，后续聊天中有新决定时再滚动回填。

使用时机：

- 启动一本新书 / 新项目时；
- 用户准备开始阅读理解、切集、画本或录制版生产前；
- 接手一个中途项目，但制作参数没有可回查记录时；
- 平台、单集时长、前情提要、声音产物形态、声音处理强度、CV 分配、彩色录制版等关键参数发生变化时。

执行原则：

- 不要求用户一次性想全所有细节；
- 不用大问卷打断工作；
- 只追问会阻塞下一步的最小必要问题；
- 可给“建议默认值”，但必须标明不是制作人已确认；
- 后续确认的新信息应回填到同一份初始化记录。

### 最小字段

| 字段 | 用途 |
|---|---|
| 书名 | 所有产物标题 |
| 类型 / 题材 | 标签扩展、演播风格 |
| 目标平台 | 单集时长、标题规则、审核要求 |
| 单集目标成品时长 | 集数规划基准 |
| 片头时长 | 成品时长公式 |
| 是否需要前情提要 | 时长和画本结构 |
| 前情提要模式 | 固定 / 每集变量 |
| 前情提要目标时长 | 正文目标时长扣除项 |
| 前情提要是否进入录制文本 | 是否使用 `【前情提要】` |
| 片尾时长 | 成品时长公式 |
| 初始朗读速度 | 220–280 字/分钟粗估 |
| 声音产物形态 | 有声书 / 多人有声书 / 轻广播剧化 / 广播剧或声音剧 / 待确认 |
| 声音处理强度 | 不处理 / 原文画本 / 录制执行稿 / 轻广播剧化 |
| 下一步生产动作 | 先原文画本 / 先录制执行稿 / 进入录制版 / 停在当前产物 / 待确认 |
| 是否需要正式 CV 分配 | 决定录制责任颗粒度 |
| 是否需要彩色录制版 | 决定是否派生 Word/WPS 画本 |
| 平台要求 | 标题、审核、合同、禁忌 |
| 当前状态 | 已确认 / 待确认 / 建议默认值 |
| 回填记录 | 后续确认或修改的日期与内容 |

缺项可标“待确认”；只追问会阻塞下一步的最小必要问题。

## 5. 输入形态判断

### 已拆分集

- 尊重现有集边界。
- 阅读理解块可覆盖 2–3 个完整集。
- 不轻易拆开单集。

### 未拆分原文

- 先按 3000–5000 字自然剧情边界形成阅读理解块。
- 读懂后再做集数规划。
- 不机械按章节切集。

## 6. 分集 / 拆章语义完整性规则

拆章不是按字数机械切分，而是为听众重新设计自然停顿点。每一集应尽量成为一个可独立进入、可顺畅理解、可自然暂停的听觉单元。

### 6.1 判断优先级

分集时按以下优先级判断：

1. 听众理解顺滑度；
2. 剧情 / 论述 / 情绪链条完整性；
3. 单集时长与字数目标；
4. 标题吸引力与平台节奏。

字数和时长是项目参数，不是硬性切刀。目标区间应根据平台、语速、内容类型和成品时长另行设定。当目标字数与语义完整冲突时，优先保留完整语义块，并允许合理浮动。

### 6.2 章首规则

新一集的开头应尽量从新的时间、地点、动作、事件、场景或议题入口开始。章首应让听众即使隔天继续听，也能迅速进入当前内容，而不是被迫回忆上一集最后一句。

如果某一句必须依赖上一集最后一句才能听懂，或本质上是在回应、解释、总结、补充上一场景，那么它不应作为新集开头，应回收到上一集。

不适合作为章首的类型包括：

- 对上一句的直接回复；
- 短信、电话、聊天中的单句回应；
- 问答链条中的回答或追问；
- 上一场景后的心理落点；
- 上一场景的动作收束；
- 上一段内容的解释、总结、反讽或补充说明；
- 清单、信件、记录、长段引用中的中段内容；
- 争执、谈判、告白、摊牌等高连续性场景的中段。

### 6.3 章尾规则

章尾应停在听众可以自然暂停的位置，而不是停在字数刚好到达的位置。

适合作为章尾的位置包括：

- 一个小事件完成后；
- 一轮对话或问答闭合后；
- 一个行动产生结果后；
- 一个发现、决定、反转或认知变化落下后；
- 一个情绪波峰或情绪落点完成后；
- 一个新悬念已经建立，但前因交代清楚时。

章尾可以制造继续收听的欲望，但不能把“前因”留在上一集、“回应 / 结果 / 心理落点”扔到下一集开头。好的悬念是“接下来会怎样”，不是“刚才为什么这样”。

### 6.4 完整语义块保护

以下内容必须优先作为完整语义块处理，不应为了凑字数或时长硬切。除非该语义块本身过长到严重影响平台时长，否则应整体保留在同一集内。

- 一轮完整对话或问答；
- 一组连续短信、聊天、电话、语音往返；
- 一次争执、谈判、告白、摊牌、审问、对峙；
- 一封信、一篇日记、一份记录、一段备忘、一组引用材料；
- 一份清单、目录、账目、证据列表、行动列表；
- 一组连续编号内容；
- 一个作品、方案、计划、报告、画作、文章、项目等被逐项展开的结构；
- 一个象征物、关键道具、意象或反复出现的线索段落；
- 一个动作场景的起因、过程、结果；
- 一个观点的提出、论证、结论；
- 一个角色从外部事件到内心反应的完整链条；
- 一段“发现问题 → 记录问题 → 得出结论”的自我整理过程；
- 一段“准备 → 执行 → 完成 / 失败 / 反转”的行动闭环。

尤其要保护“发生了什么 → 角色如何回应 → 这件事在角色心里意味着什么”这条链条。心理落点、认知变化和情绪收束，通常属于上一场景的完成部分，不宜切到下一集开头。

如果文本中出现连续编号、分项展开、清单式记录、阶段性总结、作品结构拆解等内容，应先判断它是否构成一个整体表达单元。若构成整体，则不能只按字数切开其中一部分。

### 6.5 标题规则

标题不是剧情标签，而是听众入口。标题应帮助听众理解这一集的核心吸引点，并产生继续点击的欲望。

标题应优先提炼：

- 本集的关系真相；
- 本集的情绪刺点；
- 本集的认知反转；
- 本集的人物觉醒；
- 本集的选择代价；
- 本集的核心矛盾；
- 本集最有记忆点的判断或发现。

标题不宜只是直白复述剧情，例如“她做了某事”“他说了某话”“某人来了”“两人吵架了”。这类标题对未听过正文的听众吸引力弱，也容易显得流水账。

好的标题应满足：

- 对应本集真正完成的核心推进；
- 不错位到上一集或下一集的内容；
- 不提前剧透关键反转到削弱收听欲；
- 不为了金句感而脱离剧情；
- 即使没听过正文，也能让听众获得一点判断、疑问或情绪牵引；
- 尽量短、有力、有辨识度；
- 能体现作品原有气质，而不是统一套用营销标题腔。

标题可以有“金句感”或“醍醐灌顶感”，但必须从本集内容自然提炼出来。标题的作用不是替正文总结主题，而是为听众打开这一集。

### 6.6 分集复审

拆章完成后，必须逐集复审章首、章尾、语义块和标题，不能只检查字数表。每一集至少检查：

- 章首是否能独立进入；
- 章首是否依赖上一集最后一句；
- 章首是否属于上一场景的回复、反应、解释或心理落点；
- 章尾是否收住了当前语义块；
- 是否切断了对话、短信、电话、信件、清单、记录、争执、谈判等连续内容；
- 是否切断了连续编号、分项展开、作品结构、象征线索或行动闭环；
- 是否把“事件发生—角色回应—心理理解”拆成了两集；
- 标题是否对应本集核心推进；
- 标题是否只是剧情复述；
- 标题是否和上一集或下一集内容错位；
- 标题是否对未听过正文的听众也有吸引力。

一句话原则：分集拆章的本质，是替听众安排“在哪里进入”和“在哪里停下”。每一集都应做到：进得来、听得懂、情绪不断、意思完整、愿意继续。

## 7. 产物分层

| 层级 | 产物 |
|---|---|
| 理解层 | 阅读理解块记录、全书资料包、角色分析工作簿、原文问题与制作修订记录 |
| 规划层 | 项目初始化记录、集数规划记录、角色颜色与录制责任记录 |
| 原文画本层 | 原文画本 |
| 声音执行层 | 录制执行稿 |
| 录制层 | 录制版画本、试录校准记录、补录清单、角色工作稿 |

所有关键理解必须落地为可回查产物，同时服务后续 AI 工作和制作人决策。

## 8. 最小试跑闭环

1. 项目初始化记录；
2. 1–2 个阅读理解块记录；
3. 全书资料包草稿 / 增量；
4. 角色分析工作簿草稿；
5. 第 1 集集数规划记录；
6. 角色颜色与录制责任记录草稿；
7. 第 1 集下一步生产动作判断；
8. 按路径生成第 1 集原文画本 / 录制执行稿 / 录制版画本中的必要产物；
9. 如已进入录制，生成试录 / 第 1 集校准记录。

## 9. 分支规则

### 原文画本与录制执行稿

- 可在集数规划完成后直接从分集文本进入原文画本。
- 可在集数规划完成后直接从分集文本进入录制执行稿。
- 也可在原文画本之后基于画本生成录制执行稿。
- 已完成原文画本不排斥后续录制执行稿；已完成录制执行稿通常不需要再倒做原文画本，除非制作人需要忠于原文的结构母版。
- 如果本轮目标只是结构标注，可停在原文画本。
- 如果本轮目标是听觉友好、角色拆分、表演提示、音效配乐或后期可执行，可进入录制执行稿。
- 不需要：跳过对应层。

进入录制执行稿前，应先判断声音产物形态：

- 有声书 / 多人有声书：进入默认录制执行稿规则。
- 轻广播剧化：可进入录制执行稿，但必须保留原文叙事骨架，只扩大声音化尺度。
- 广播剧 / 声音剧：不由当前录制执行稿规则默认生成，应另设广播剧剧本或声音剧执行规则。

录制执行稿必须与原文整理、分集文本、原文画本和录制版画本物理隔离。

### CV / 录制责任

- 未定 CV 不阻塞原文画本、录制执行稿或录制版画本的前期准备。
- 颜色绑定角色，不绑定 CV。
- 录制版阶段按需显示录制责任。
- 小团队可轻量处理，只要录制责任状态明确。
