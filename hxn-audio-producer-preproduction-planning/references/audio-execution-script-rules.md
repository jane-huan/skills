# Audio Execution Script Rules

> Version: v0.6.9
> Status: 正式发布，持续迭代

## 1. 定位

录制执行稿是把分集文本或原文画本转换成可录、可演、可剪、可制作的声音执行稿。当前默认模式是有声书 / 多人有声书录制执行稿。

它不是单纯加标签，也不是重写剧情；它在不改变原文事实、人物关系和主线的前提下，把原文已有的信息、情绪、动作、潜台词和节奏，转化成演员、后期和制作人都能执行的声音指令。

轻广播剧化是录制执行稿的扩展尺度，不等同于广播剧。广播剧 / 声音剧需要另设剧本或声音剧执行规则，不由本规则默认生成。

一句话原则：

```text
信息不丢，剧情不变，听感更顺，表演更清楚，后期更可剪。
```

## 2. 与其他产物的边界

| 产物 | 作用 |
|---|---|
| 分集文本 | 确定本集原文范围、标题、起止句、字数、时长参数和切集依据 |
| 原文画本 | 忠于原文的结构标注母版，标明旁白、角色、心声、系统音、群体声、特殊声、场景和必要音效 |
| 录制执行稿 | 完成听觉友好改写、声音单元拆分、承接方判断、表演提示、音效/配乐提示和待确认项 |
| 录制版画本 | 给录制者 / 制作人使用的最终呈现版，加入颜色、CV / 录制责任、兼任提示、排版、补录和校准区 |

如果用户要求“听起来顺”“可录制”“每句角色拆分”“参考语气”“音效配乐”“后期提示”“短旁白转 CV”“文字载体怎么处理”，应进入录制执行稿，而不是只生成原文画本。

## 3. 核心处理顺序

1. 读懂本集：确认场景、人物、情绪链、信息链和需要保护的原文事实。
2. 确认声音产物形态：有声书 / 多人有声书为默认；轻广播剧化只扩大声音化尺度；广播剧 / 声音剧转入专门规则。
3. 判断听觉障碍：代词不清、承接跳跃、文字载体不可听、心理转折不明、场景变化不清。
4. 判断信息承接方：CV、旁白、心声、音效、配乐、特殊声、引用或留白。
5. 拆成声音单元：按听感和制作可执行性组织，不追求标签越多越好。
6. 添加必要执行提示：参考语气、音效、配乐、停顿、待确认项。
7. 做信息回查：确认没有新增剧情事实、没有丢失状态/动作/空间/心理/文字信息。

## 4. 可调整与禁止

允许调整：

- 代词、语序、台词切分；
- 必要角色名、关系信息、动作承接或场景转场；
- 短状态从旁白转入 CV 括号提示；
- 文字载体的声音呈现方式；
- 音效、配乐、停顿和参考语气的位置。

禁止调整：

- 新增原文没有的事件；
- 改变人物关系或主线；
- 把暗示改成确定剧情事实；
- 替角色补出原文没有的明确动机；
- 用音效、配乐或表演提示替代必要剧情信息。

## 5. 信息承接判断

先问：这个信息最终由谁承接最自然？

| 信息类型 | 常见承接方式 |
|---|---|
| 事件、空间、时间、逻辑 | 旁白 / 场景 |
| 台词、状态、潜台词、短动作感 | CV |
| 具体动作、空间变化、节奏打断 | 音效 / 旁白 |
| 情绪底色、段落压力、剪辑结构 | 配乐 |
| 未说出口的想法、认知转折 | 心声 / 旁白 / 停顿 |
| 消息、通知、记录、规则、清单、社交平台内容 | 旁白 / 角色声 / 特殊声 / 引用 / 音效辅助 |

来源属于谁，不等于一定由谁发声；最终由谁承接，以听感清楚、执行自然为准。

## 6. 有声书录制执行稿的承接方防误判

有声书 / 多人有声书模式下，旁白通常是叙述、解释、视角、关系判断和场面状态的安全承接通道。旁白默认安全不等于永远旁白优先；只有当角色声、特殊声、群体声或音效明确提升理解、表演或剪辑时，才拆出其他声源。

可声音化不等于应声音化。录制执行稿的目标是可录、可懂、可剪，不是把信息尽量分配给更多声源。

群体声只承接明确可发声的群体内容；叙述性群体反应、场面气氛或关系判断，优先由旁白、音效或节奏提示承接。

文字载体先判断内容呈现是否清楚，再判断是否需要保留发出者责任。清楚优先于媒介花样；责任归属只在影响理解、剧情或表演时强化。

同一角色对白后紧接心声时，应防止听众误判为对白延续。若缺少明确边界，优先由旁白承接，或加入停顿、音效、混响、标签等隔离提示。

每次分配声源后都要检查：听众能否分清这是说出口的话、心里想法、旁白判断、文字内容还是现场反应。

## 7. 夹叙信息的听觉连续性与声音承接

录制执行稿处理夹在对白中的叙述信息时，首要目标是让听众持续进入场景，而不是把所有信息机械拆分。小说中的夹叙可能包含动作、神态、心理、语气、停顿和关系反应；改写时应先判断这段夹叙在听觉上是否会打断对话节奏，再决定由旁白、CV、音效或参考语气承接。

处理夹叙信息时，先判断它属于哪一类：

- 完整叙述段：如果夹叙本身承担动作推进、关系反应、心理变化或场景落点，即使其中同时包含可听状态、视觉动作和心理信息，也应优先作为旁白整体保留或适度整合。必要时可以优化句式，让旁白更顺耳，但不能因为其中有可由 CV 表演的状态，就把整段拆散。
- 短声音状态：如果夹叙非常短，且主要描述可由声音自然呈现的状态，例如轻笑、冷笑、叹息、哽住、声音变轻、语气放软、短暂停顿、迟疑等，可以优先转入目标台词括号，或写成紧贴目标台词上方的参考语气，避免对话被三两个字的旁白频繁打断。
- 纯视觉动作或神态：例如皱眉、点头、低头、转身、抬眼、手一抖、眼神变化等，不能直接塞进 CV 台词括号。若它只是短促状态，且不承担重要剧情或心理功能，可以视听感需要省略或融入参考语气；若它承担动作推进、关系反应、心理变化或节奏落点，应由旁白保留或与相邻叙述合并承接。
- 空间、动作或沉默压力：如果能用音效增强空间、节奏或沉默压力，可以交给音效辅助，但不能用音效替代必须让听众听懂的剧情信息。

录制执行稿的目标不是减少旁白，而是按听觉连续性选择最自然的承接方式：短而可演的交给 CV，完整叙述交给旁白，能增强空间或节奏的交给音效，心理和关系转折必须保留清楚。

避免两种问题：

- 为了顺耳删掉原文中承担状态、动作、心理或关系压力的信息。
- 为了忠实原文，把所有夹叙逐条拆成零碎旁白，导致对话不断被打断。

正确做法是在不改剧情事实的前提下，把叙述信息转成更适合听觉接收的声音单元。

## 8. 短旁白转 CV 提示

紧贴台词的短状态信息，如果单独保留为旁白会破坏听感，可以转移到角色台词括号提示中，由 CV 表演承接。

短旁白转 CV 只适用于短而可由声音自然呈现的状态，不适用于纯视觉动作、完整叙述段、动作推进或心理转折。

```text
原文：
角色名 + 短状态 / 短动作 + ：“台词。”

可改为：
【角色名】“（短状态 / 表演提示）台词。”
```

如果该信息承担动作推进、场景变化、节奏停顿或剧情功能，不能吞进台词，应由旁白、场景或音效承接。

## 9. 表演与节奏提示

参考语气 / 表演提示必须放在目标句上方，不要放在目标句下方或远离目标句的位置。一个提示只控制紧随其后的目标句或目标句组；如果控制一组台词，应在提示里写明范围。

不要只写情绪词。参考语气 / 表演提示应尽量说明：

- 表面听感；
- 台词功能；
- 情绪边界；
- 重音、停顿或节奏；
- 避免演成什么。

台词功能可按文本判断，例如推进、打断、确认、回避、试探、安抚、施压、转移、示弱、拒绝、求助、掩饰、切断关系、制造期待等。不要固定套用清单。

普通叙事、清楚台词、无需额外说明的句子，不加参考语气。

旁白节奏提示也应放在目标旁白上方，用于控制停顿、压低、加速、放慢、留白或段落转折。不要把旁白节奏提示混进旁白正文，除非它本身就是要读出的内容。

## 10. 文字载体

消息、通知、备注、规则、清单、记录、邮件、社交平台内容等文字载体，不必一律标为 `引用`。

可选：

```text
【旁白】文字内容。
【特殊声：媒介类型】文字内容。
【引用】文字内容。
【角色名】“文字内容。”
```

选择标准：

- 旁白读最清楚：旁白；
- 需要媒介感：特殊声；
- 需要保留书面感：引用；
- 需要角色责任或来源：角色声。

先解决听众如何听懂内容，再决定是否突出文字来源或发出者责任。不要为了媒介感牺牲清楚度。

## 11. 声音连续性与段落收束

生成录制执行稿时，先判断声音连续性，再决定标签分段和声源承接，不把收束问题留到事后清理。

连续旁白不要因原文自然分段而机械切碎。同一动作、同一情绪流或同一场景状态下，可按听觉连续性合并；出现情绪 / 认知转折、动作 / 时间 / 场景变化，或段落过长影响换气、录制和定位时，再自然拆开。

记录性文字不要逐句逐字段拆标签。日记、备忘、清单、记录、日志、复盘等，应先按语义合并为可录段，再判断由旁白、角色声、心声或特殊声承接。

不要为了标签整齐切碎旁白或记录，也不要为了合并而吞掉节奏、转折和必要表演提示。

## 12. 转场与场景进入提示

转场不只是时间地点切换。必要时可以用一句简短提示说明新场景中的人物状态、关系张力、情绪对比或表演入口，帮助旁白、CV 和后期快速进入下一场。

转场提示应来自原文已有信息和上下文判断，不能补出原文没有的人物动机、事件或因果。它是执行说明，不是新增剧情。

可选格式：

```text
【转场：新场景入口；人物状态 / 关系张力 / 情绪对比 / 后期衔接方向】
【场景】时间 / 地点 / 情境变化。
```

如果只是清楚的时间地点切换，不必额外添加转场说明；如果转场本身承担情绪落差、关系压力或节奏重启，应写出可执行入口。

## 13. 音效与配乐

音效必须具体、可剪辑，服务动作、空间、转场、心理节奏、信息打断或沉默压力。避免只写“氛围音”。

音效尽量写明：

- 具体声源；
- 出现位置；
- 轻重关系；
- 持续方式。

配乐必须有边界，尽量写明：

- 进入点；
- 持续范围；
- 淡出点；
- 情绪功能；
- 音量关系；
- 素材搜索方向。

必要时可以补充简短素材检索关键词，帮助后期快速找到可用素材。关键词应服务剪辑执行，例如情绪方向、节奏速度、器乐质感、场景类型或悬疑 / 温情 / 压迫等素材库常用检索方向。

配乐服务段落情绪和剪辑结构，不替代剧情表达。配乐提示要简洁，不写成文学赏析，不替代剧情和表演。

## 14. 推荐格式

默认按以下顺序组织，但格式服从听感和制作可执行性：

```text
章标题
【转场：新场景入口；人物状态 / 关系张力 / 情绪对比 / 后期衔接方向】
【场景】时间 / 地点 / 情境变化。
【音效：具体声源 / 动作声 / 环境声 / 持续方式】
【配乐：进入点、持续范围、淡出点、情绪功能、音量关系、素材搜索方向】
【参考语气】放在目标台词上方；说明表面听感、台词功能、情绪边界、重音 / 停顿、避免演成什么。
【角色名】“台词。”
【旁白节奏】放在目标旁白上方；说明停顿、压低、加速、放慢、留白或段落转折。
【旁白】旁白内容。
【角色名｜心声】心声内容。
【特殊声：媒介类型】文字内容。
【群体声｜处理方式】内容。
```

角色台词里的短动作或短状态可以嵌入台词括号中，由 CV 承接：

```text
【角色名】“（叹口气）台词。”
```

较长的表演说明不要塞进台词括号，应放到目标句上方的 `【参考语气】`。

## 15. 验收清单

每集完成后检查：

1. 听众只听一遍能否听懂？
2. CV 是否知道谁说、怎么说、为什么这么说？
3. 后期是否知道哪里进音效、哪里进配乐、哪里留空白？
4. 是否新增了原文没有的剧情事实？
5. 声音产物形态是否清楚，是否把广播剧 / 声音剧误用为当前默认规则？
6. 听众是否能分清说出口的话、心里想法、旁白判断、文字内容和现场反应？
7. 是否把可由旁白清楚承接的信息过度拆成多声部？
8. 群体声、特殊声、心声是否真正提升理解或执行，而不是只因为“可以声音化”才使用？
9. 连续旁白和记录性文字是否没有因机械分段被切碎？
10. 是否因删短叙述而丢失状态、动作、心理或关系压力？
11. 夹叙信息是否先按听觉连续性判断，再分配给 CV、旁白、音效、配乐或留白？
12. 完整叙述段是否没有因为含有可演状态而被拆碎？
13. 纯视觉动作或神态是否没有被直接塞进 CV 台词括号？
14. 配乐提示是否服务后期执行，而不是抽象赏析？
15. 转场提示是否只说明执行入口，没有新增剧情？
16. 是否存在标签很多但不可执行的地方？
17. 是否存在过度拆碎、反而破坏听感的地方？
