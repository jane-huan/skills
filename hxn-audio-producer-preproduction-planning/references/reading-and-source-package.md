# Reading and Source Package

> Version: v0.6.9
> Status: 正式发布，持续迭代

## 1. 阅读理解块切分

- 默认每块 3000–5000 字。
- 优先自然剧情边界：场景转换、事件结束、视角转换、章节节点。
- 已拆分集时尊重集边界，可 2–3 集为一块。
- 未拆分原文时先阅读理解，再做集数规划。
- 不为了凑字数切断强连续场景。
- 阅读块不是最终分集；阅读时只标记潜在切集点和语义风险，最终分集必须在读懂上下文后复审。

## 2. 逐块分析原则

每块只做理解记录，不做逐句演播设计。必须记录：

- 覆盖范围、起始句、结束句；
- 主要内容；
- 主要人物；
- 新增设定、伏笔、关系变化、重要物件；
- 可进入全书资料包的信息；
- 潜在切集点；
- 潜在切集点的章首 / 章尾理由；
- 不宜切断的完整语义块；
- 连续编号、分项展开、清单式记录、阶段性总结、作品结构拆解、象征线索或行动闭环等整体表达单元；
- 原文问题。

禁止：

- 脑补后续剧情；
- 把猜测写成确定设定；
- 在阅读块里做细致音效和后期设计；
- 只给摘要，不留定位信息。

## 3. 资料包滚动积累

每个阅读块结束后，更新资料包草稿中的：

- 作品概览候选信息；
- 世界观 / 设定；
- 角色画像；
- 人物关系；
- 演播风格线索；
- 前情提要素材；
- 统一称呼、易错词、禁忌；
- 原文问题与待确认项。

资料包不能只存在对话上下文里，必须落地为可回查记录。

## 4. 全书资料包 v1

生成条件：

- 已读完整本，或已读到足以支撑当前生产阶段的范围；
- 阅读理解块记录可回查；
- 重要待确认项已列出。

生成原则：

- 基于阅读块记录汇总，不凭空总结；
- 是制作人手册，不是剧情流水账；
- 角色理解要能支撑角色颜色与录制责任记录、原文画本、录制执行稿和录制版画本；
- 不确定内容标“待确认”。

## 4.1 角色分析工作簿

角色分析工作簿不是逐块阅读时即时定稿的产物，而是在完成阅读理解块、统一汇总全书阅读理解，并形成全书资料包 v1 后生成的角色专项模块。

生成原则：

- 基于阅读理解块记录、全书资料包 v1、分集文本、原文画本、录制执行稿或可回查原文统计，不凭印象补全；
- A / B / C 分类要同时参考剧情重要度、出场覆盖、台词量、关系功能、声音独立性和兼任风险；
- 年龄尽量给出区间，如 30-35 岁；无法确认时写“年龄感 + 待确认”，避免只写“都市青年”“成熟男声”等泛化描述；
- 人物经历简介以本书完整故事为单位，简短概括角色在故事里的主要经历、身份变化、处境变化和关键遭遇；它可以辅助声音和表演判断，但不需要拆成细颗粒心理分析；
- 声音建议与 CV 声线条件分开：前者描述角色听感，后者描述选角硬条件；
- 出场顺序、出现篇幅、台词段数、台词字数、原文提及次数必须注明统计口径和置信度；
- 台词统计可先用自动统计形成候选，但正式录制前应允许人工校对归属不明、群杂、旁白引用和心声等情况；
- 角色分析工作簿可作为生成“角色颜色与录制责任记录”的上游依据，但不替代制作人最终分配。

## 5. 低质量原文处理

发现错字、缺文、重复、台词归属不明、逻辑冲突、审核风险时：

1. 不直接悄悄改原文；
2. 记录到“原文问题与制作修订记录”；
3. 标记是否影响资料包、原文画本、录制执行稿、录制版画本；
4. 需要制作人判断时暂停该处定稿。
