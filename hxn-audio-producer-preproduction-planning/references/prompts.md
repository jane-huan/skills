# Prompts

> Version: v0.6.9
> Status: 正式发布，持续迭代

本文件保存音频制作人前期筹备流程中可直接复用的 AI 执行 prompt。使用时按任务选择对应 prompt，不需要一次性读取全部 prompt。

## 统一工作原则

- 新书 / 新项目启动时，必须先建立轻量项目初始化记录；缺项可待确认并后续回填。
- 阅读理解块必须留痕，不能只存在对话上下文。
- 资料包边读边积累，读完后整理为 v1。
- 片头、前情提要、片尾必须进入时长估算。
- 前情提要如果进入录制版画本，必须使用 `【前情提要】` 标签。
- 颜色绑定角色，不绑定 CV。
- 原文画本不绑定 CV；同场景 CV 兼任提示只进入录制版画本。
- 集数规划完成后先判断下一步生产动作：先原文画本、先录制执行稿、进入录制版、停在当前产物或待确认。
- 录制执行稿是可选分支，可由分集文本或原文画本直接进入；不需要时跳过。
- 全部关键产物必须落地为可回查记录：Markdown、Word/WPS、结构化表、本地文件或其他载体均可。

## Prompt 0｜轻量项目初始化记录

### 使用场景
处理正文前、启动新书 / 新项目时使用；接手中途项目但缺少制作参数记录时也使用。

### Prompt

```text
请在处理正文前，先建立本项目的“轻量项目初始化记录”。

这不是一次性填完的大问卷。请把用户已经提供的信息先落地；没有提供的信息标为“待确认”；可以给建议默认值，但必须标明不是制作人已确认。只追问会阻塞下一步的最小必要问题，不要直接生成画本。

请按以下字段整理：
1. 项目类型：纯有声书 / 多人演播 / 轻广播剧化 / 其他
2. 目标平台与单集目标成品时长
3. 片头固定时长
4. 是否需要前情提要
5. 前情提要时长模式：固定 / 每集变量
6. 前情提要目标时长
7. 前情提要是否进入录制版画本
8. 片尾固定时长
9. 正文目标时长
10. 朗读速度初始估算值
11. 是否需要试录校准
12. 声音产物形态：有声书 / 多人有声书 / 轻广播剧化 / 广播剧或声音剧 / 待确认
13. 是否需要录制执行稿以及处理强度
14. 是否需要正式 CV / 录制责任管理
15. 是否需要彩色 Word/WPS 录制版
16. 分集后的下一步生产动作：先原文画本 / 先录制执行稿 / 进入录制版 / 停在当前产物 / 待确认
17. 当前状态：已确认 / 待确认 / 建议默认值
18. 回填记录：后续确认或修改时追加

如果后续聊天中用户确认或修改了参数，请回填到同一份初始化记录，不要新建多份互相冲突的记录。
```

### 输出要求
- 输出“已确认 / 待确认 / 建议默认值”三类字段。
- 明确哪些参数会影响集数规划、原文画本、录制执行稿、录制版画本和试录校准。
- 如果没有阻塞项，明确说明“可先继续阅读理解 / 资料包 / 统计，待确认项后续回填”。

## Prompt 1｜阅读理解块分析

### 使用场景
读取长篇原文的一个自然块，沉淀可回查理解记录。

### Prompt

```text
请分析以下阅读理解块，并按字段输出“阅读理解块记录”。

字段：
- 块编号
- 输入形态：已分集 / 未分集
- 覆盖集数 / 原文范围
- 字数
- 起始句 / 结束句
- 主要内容：3～6 条
- 本块主要人物
- 新增重要信息：设定、伏笔、关系变化、重要物件
- 可进入全书资料包的信息
- 潜在切集点
- 原文问题
- 是否建议更新资料包草稿

要求：
- 只基于给定文本，不脑补后文。
- 不做逐句演播设计。
- 不把临时猜测写成确定设定。
- 对后续 AI 或制作人有复用价值的信息，要单独列入“可进入全书资料包的信息”。
```

### 输出要求
- 优先用表格。
- 如果发现原文问题，必须注明位置和影响范围。

## Prompt 2｜全书资料包 v1 生成

### 使用场景
基于多个阅读理解块记录，生成可服务制作人、AI、CV 和画本生产的资料包。

### Prompt

```text
请基于所有阅读理解块记录，整理“全书资料包 v1”。

结构必须包括：
1. 作品概览
2. 全书梗概
3. 世界观 / 基础设定
4. 主要角色画像
5. 人物关系
6. 演播风格总说明
7. 重要注意事项
8. 前情提要规则
9. 原文问题与制作修订入口

要求：
- 资料包是制作人手册，不是剧情流水账。
- 角色理解要能支撑角色颜色与录制责任记录、原文画本、录制执行稿和录制版画本。
- 只使用阅读理解块中已经确认的信息。
- 不确定的信息标为“待确认”，不要补设定。
```

### 输出要求
- A/B/C 角色分层清楚。
- 统一称呼、易读错词、审核风险、不能改动的设定要单列。

## Prompt 2.1｜角色分析工作簿生成

### 使用场景
AI 完成阅读理解块、统一汇总全书阅读理解，并形成全书资料包 v1 后，生成角色拆解、人物画像、剧情对照、台词统计和选角兼任建议。

### Prompt

```text
请基于阅读理解块记录、全书资料包 v1、集数规划记录、原文画本、录制执行稿或可回查原文统计，生成“角色分析工作簿”。

必须输出 5 张表：
1. 01_角色总表
2. 02_剧情对照
3. 03_台词统计
4. 04_选角兼任
5. 05_待确认项

01_角色总表字段：
- 角色名
- 别名 / 称呼
- 角色档位：A / B / C
- 角色类型
- 性别
- 年龄区间
- 身份 / 职业 / 阵营
- 人物定位
- 人物经历简介
- 核心目标 / 欲望
- 性格 / 气质
- 情绪底色
- 说话方式
- 关系核心
- 人物弧光 / 变化
- 剧情功能
- 声音建议
- CV 声线条件
- 演播注意事项
- 颜色建议
- 制作优先级
- 证据来源
- 待确认项

02_剧情对照字段：
- 角色名
- 角色档位
- 首次出场
- 最后出场
- 出场顺序
- 覆盖集数 / 篇幅
- 关键阶段
- 关键事件
- 关系变化
- 情绪变化
- 声音阶段提示
- 关联阅读块

03_台词统计字段：
- 角色名
- 角色档位
- 出场顺序
- 首次出场
- 出现的篇幅
- 出现集数
- 台词段数
- 台词字数
- 原文提及次数
- 旁白承载量
- 统计口径
- 统计来源
- 置信度
- 统计备注

04_选角兼任字段：
- 角色名
- 角色档位
- 是否独立声线
- 是否建议单独录制
- CV 声线条件
- 可否兼任
- 建议兼任方向
- 同场景兼任风险
- 风险依据
- 试音优先级
- 试音片段建议
- 录制责任人 / CV
- 备注

05_待确认项字段：
- 项目
- 角色名
- 问题
- 影响产物
- 建议处理
- 状态
- 备注

要求：
- 只能使用已完成阅读理解、资料包和可回查统计中的信息，不脑补原文。
- 年龄尽量给区间；无法确定时写“年龄感 + 待确认”，不要只写泛化标签。
- 人物经历简介以本书完整故事为单位，简短概括角色在故事里的主要经历和处境；可以辅助声音判断，但不需要拆成细颗粒心理分析，也不写无关生平流水账。
- “声音建议”和“CV 声线条件”必须分开。
- 台词段数、台词字数、出现篇幅、原文提及次数必须写明统计口径、来源和置信度。
- A/B/C 分类同时考虑剧情重要度、出场覆盖、台词量、声音独立性和兼任风险。
- 无对白但剧情存在感强的角色，要在旁白承载量和演播注意事项中说明。
- 所有不确定信息进入 05_待确认项。
```

### 输出要求
- 优先输出 Markdown 表格；如需 Excel，可按 5 张表生成工作簿。
- Excel 版建议冻结首行、开启筛选、长文本自动换行，并用角色档位、风险等级、优先级、待确认状态做颜色区分。
- 该工作簿可作为“角色颜色与录制责任记录”的上游输入，但不替代制作人最终确认。

## Prompt 3｜未分集原文的集数规划

### 使用场景
原文尚未按音频集数切分，需要根据项目参数、阅读理解记录和资料包做生产切集。

### Prompt

```text
请根据项目初始化参数、阅读理解块记录和全书资料包，生成“集数规划记录”。

每集必须计入：
- 集号
- 集标题
- 原文范围
- 起始句 / 结束句
- 正文字数
- 核心事件
- 结尾钩子
- 章首独立性检查
- 章首风险
- 章尾完整性检查
- 被保护语义块
- 整体表达单元检查
- 语义切断风险
- 标题贴合度
- 标题吸引力检查
- 标题风险
- 单集目标成品时长
- 片头时长
- 前情提要时长模式
- 前情提要目标时长
- 片尾时长
- 正文目标时长
- 当前朗读速度
- 预计正文时长
- 预计成品时长
- 时长偏差等级
- 原文画本状态
- 录制执行稿状态
- 下一步生产动作
- 录制版状态

要求：
- 分集不是按字数机械切分，而是为听众设计自然进入点和自然暂停点。
- 判断优先级依次为：听众理解顺滑度；剧情 / 论述 / 情绪链条完整性；单集时长与字数目标；标题吸引力与平台节奏。
- 字数和时长是项目参数，不是硬性切刀；当目标字数与语义完整冲突时，优先保留完整语义块，并允许合理浮动。
- 新一集开头应尽量从新的时间、地点、动作、事件、场景或议题入口开始。
- 不要把对上一句的直接回复、短信 / 电话 / 聊天单句回应、问答链条中的回答或追问、上一场景后的心理落点、动作收束、解释总结、反讽补充、清单 / 信件 / 记录中段、高连续性场景中段作为章首。
- 章尾应停在可自然暂停的位置：小事件完成、对话闭合、行动产生结果、发现 / 决定 / 反转 / 认知变化落下、情绪波峰或落点完成、新悬念已建立且前因交代清楚。
- 可以制造“接下来会怎样”的悬念，不能把“刚才为什么这样”的前因、回应、结果或心理落点切到下一集开头。
- 一轮完整对话或问答、一组连续短信 / 聊天 / 电话 / 语音往返、一次争执 / 谈判 / 告白 / 摊牌 / 审问 / 对峙、一封信 / 日记 / 记录 / 备忘 / 引用材料、一份清单 / 目录 / 账目 / 证据列表 / 行动列表、一组连续编号内容、一个作品 / 方案 / 计划 / 报告 / 画作 / 文章 / 项目的逐项展开结构、一个象征物 / 关键道具 / 意象 / 反复出现的线索段落、一个动作场景的起因过程结果、一个观点的提出论证结论、一个角色从外部事件到内心反应的链条、一段“发现问题 → 记录问题 → 得出结论”的自我整理过程、一段“准备 → 执行 → 完成 / 失败 / 反转”的行动闭环，应优先作为完整语义块处理。
- 尤其保护“发生了什么 → 角色如何回应 → 这件事在角色心里意味着什么”的链条；心理落点、认知变化和情绪收束通常属于上一场景的完成部分，不宜切到下一集开头。
- 如果文本中出现连续编号、分项展开、清单式记录、阶段性总结、作品结构拆解等内容，先判断它是否构成一个整体表达单元；若构成整体，不能只按字数切开其中一部分。
- 标题不是剧情标签，而是听众入口；标题应提炼本集的关系真相、情绪刺点、认知反转、人物觉醒、选择代价、核心矛盾或最有记忆点的判断 / 发现。
- 标题不宜只是直白复述剧情，不应错位到上一集或下一集，不应提前剧透关键反转，不应为了金句感脱离剧情，也不应统一套用营销标题腔。
- 标题应短、有力、有辨识度，即使没听过正文，也能让听众获得一点判断、疑问或情绪牵引，并体现作品原有气质。
- 片头、前情提要、片尾必须进入成品时长估算。
- 如果时长偏差超过阈值，必须标出并给出处理建议。
- 拆章完成后必须逐集复审章首、章尾、语义块和标题，不能只检查字数表。
```

### 输出要求
- 输出结构化表格。
- 每集必须有听觉钩子或说明为什么本集没有明显钩子。
- 每集必须给出章首 / 章尾复审结论。
- 每集必须给出标题判断：是否对应本集核心推进、是否只是剧情复述、是否内容错位、是否对未听过正文的听众有吸引力。
- 如存在语义切断风险，必须给出调整建议：前移、后移、合并、拆出独立短集或提交制作人确认。

## Prompt 4｜逐集原文画本

### 使用场景
基于本集原文和资料包生成原文画本母版。

### Prompt

```text
请基于本集原文、全书资料包、角色颜色与录制责任记录和本书标签规则，生成逐集原文画本。

要求：
- 忠于原文，不擅自改剧情。
- 不主动做听觉友好改写，不默认加入参考语气、配乐设计或复杂后期提示。
- 如果前情提要进入画本，使用 `【前情提要】`。
- 正文开始处使用 `【正文开始】`。
- 区分 `【旁白】`、`【角色名（情绪/状态）】`、`【角色名｜心声（情绪/状态）】`、`【群体声｜处理方式（状态）】`、`【特殊声具体类型】`、`【场景：...】`、`【音效：...】`、`【引用】`。
- 群体声按需标注处理方式：多人同录 / 单人兼任 / 后期叠加 / 音效替代 / 待定。
- 可以引用角色色标，但不绑定 CV。
- 待确认项明确标出。

禁止：
- 不要为了演播效果擅自改剧情。
- 不要把 CV 名写入原文画本正文。
- 不要做最终排版。
```

### 输出要求
- 使用原文画本模板结构。
- 结尾必须包含“本集待确认问题”。

## Prompt 5｜角色颜色与录制责任记录辅助提取

### 使用场景
从资料包或已读文本中整理角色颜色、声线建议和录制责任占位。

### Prompt

```text
请基于全书资料包提取“角色颜色与录制责任记录”候选项。

字段包括：
- 角色名
- 角色档位：A / B / C
- 角色类型：主角 / 配角 / 龙套 / 群体 / 系统音等
- 简短说明
- 声线建议
- 颜色编号
- 颜色名
- HEX
- 颜色优先级
- 是否独立颜色
- 共用颜色组
- 可否兼任
- 录制责任人 / CV（可选）
- 是否独立声线 / 单独录制
- 同场景兼任风险
- 兼任提示
- 备注

原则：
- 颜色绑定角色，不绑定 CV。
- 核心角色和高频角色优先独立颜色。
- 低频角色可共用颜色。
- 录制责任人 / CV 可暂时为空，但状态要明确。
```

### 输出要求
- 优先表格。
- 不确定颜色可给候选，不要伪装成制作人已确认。

## Prompt 6｜分集后生产动作与录制执行稿

### 使用场景
集数规划完成后，判断本项目 / 本集下一步是先原文画本、先录制执行稿、进入录制版、停在当前产物，还是暂缓。

### Prompt

```text
请先判断本项目 / 本集在分集完成后的下一步生产动作，并确认声音产物形态。

动作选择：
- 先原文画本：基于分集文本生成忠于原文的结构标注母版；完成后可停在当前产物，也可继续进入录制执行稿或录制版。
- 先录制执行稿：基于分集文本直接生成可录、可演、可剪、可制作的声音执行稿；完成后可停在当前产物，也可继续进入录制版。
- 承接原文画本生成录制执行稿：已有原文画本时，基于画本继续做听觉友好、承接方判断、夹叙信息处理、参考语气、转场、音效和配乐等处理。
- 直接进入录制版：已有足够画本或文本基础，可生成录制版画本。
- 停在当前产物：当前原文画本或录制执行稿已经满足本轮共创 / 平台上传目标。
- 暂缓：缺少平台要求、声音处理方向或制作人确认。

如果执行录制执行稿，请读取 `references/audio-execution-script-rules.md`，并输出：
1. 下一步生产动作
2. 输入底稿
3. 声音产物形态：有声书 / 多人有声书 / 轻广播剧化 / 广播剧或声音剧 / 待确认

6. 承接方式说明：直接对白、夹叙信息、状态、动作、空间、心理、文字载体、转场和情绪底色分别由谁承接；说明可判定说话人的直接对白是否已拆为角色声源，夹叙是否为完整叙述段、短声音状态、纯视觉动作 / 神态，以及选择该承接方式的理由

8. 与分集文本、原文画本或输入底稿的差异说明
9. 验收结果
10. 是否到此告一段落，或是否继续进入录制版
11. 待制作人确认项

要求：
- 不得改变主线剧情。
- 不得新增未经授权的关键设定。
- 改动应可回溯分集文本、原文画本或输入底稿。

- 可声音化不等于应声音化；旁白、群体声、特殊声、心声和文字载体的承接方式应以听感清楚、声源不误解、制作可执行为准。
- 凡原文中存在明确说话人、可判定说话人，或可由上下文稳定推断说话人的直接对白，默认拆为对应角色声源并明确角色名与台词内容；不能因为对白嵌在叙述句中、夹带动作 / 神态 / 语气描写或未单独成段，就整句并入旁白。
- 连续旁白和记录性文字先按声音连续性收束，不因原文分段或字段结构机械切碎。

- 配乐和转场提示应服务录制、表演和后期执行，不写成文学赏析，不新增剧情。
- 录制执行稿必须与原文整理、分集文本、原文画本和录制版画本物理隔离。
```

## Prompt 9｜试录校准与时长偏差检查

### 使用场景
第 1 集或样张集录制完成后，用实际数据校准朗读速度。

### Prompt

```text
请根据第 1 集 / 样张集实际录制数据进行朗读速度校准。

输入：
- 正文字数
- 实际成品时长
- 实际片头时长
- 实际前情提要时长
- 实际片尾时长

计算：
实际正文时长 = 实际成品时长 - 实际片头时长 - 实际前情提要时长 - 实际片尾时长
校准朗读速度 = 正文字数 ÷ 实际正文时长

然后用校准朗读速度重新估算后续集数，并标记：
- ±10% 内：正常
- ±10%～15%：标记
- ±15%～25%：需制作人确认
- 超过 ±25%：必须重新评估切集

请输出：
1. 实际正文时长
2. 校准朗读速度
3. 与目标时长偏差
4. 偏差等级
5. 后续集数调整建议
6. 需要制作人确认的问题
```

## 输出入表格式约定
